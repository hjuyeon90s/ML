머신러닝 프로젝트 과정

1. 문제를 정의한다.
   - 목적을 명확히 정의하는 게 중요하며, 해결하려는 문제에 대한 가설을 세우고 무엇을
     해야 할지도 명확하게 해야 한다.
   - 구체적이지 못한 목표는 좋지 않은 문제 설정의 예이다.

2. 머신러닝을 사용하지 않는 방법은 없는지 검토한다.
   - 머신러닝을 꼭 사용해야 하는지를 검토한다.
   - 머신러닝을 사용하는 시스템을 구축할 때는 다음과 같은 어려움을 겪을수 있다.
     1) 확률적인 부분이 있어 자동 테스트가 어렵다.
	 2) 오래 운용하면 사용자 경향이 변해 입력 경향이 바뀐다.
	 3) 처리 파이프라인이 복잡해진다.
	 4) 데이터 의존관계가 복잡해진다.
	 5) 실험 코드 혹은 파라미터가 포함되기 쉽다.
	 6) 개발 시스템과 운용 시스템 간의 언어/프레임워크가 제각각이기 쉽다.
   - 어떤 비지니스 문제에 머신러닝을 적용하면 좋을까?
     1) 대량의 데이터에 대해 고속으로 안정된 판단을 내려야 한다.
	 2) 예측 결과에 일정 수준의 오류가 용인된다.
	 
3. 시스템을 설계한다.
   - 설계에서 중요한 핵심
     1) 예측 결과를 어떻게 이용할 것인가?
	 2) 예측 오류의 영향을 어떻게 흡수할 것인가?
	 
4. 사용할 알고리즘을 선택한다.
   - 과거에 비슷한 문제를 어떻게 풀었는지 조사해보면 대강의 후보를 정할 수 있다.
   
5. 특징과 정답 데이터, 로그를 설계한다.
   - 알고리즘 후보를 골랐다면 어떤 정보를 사용할지도 설계한다.
   - 특징( feature )은 머신러닝의 예측 모델에 입력될 정보다.
     머신러닝에서는 입력 정보를 수치 벡터로 만들어야 한다.
   - 특징( feature )을 정한 다음에는 입력 데이터로 쓸 정답 데이터( label )를 준비한다.
   
6. 데이터를 전처리한다.
   - 데이터 전처리는 해결하려는 문제에 따라 많이 달라진다.
   - 데이터 전처리는 불필요한 정보를 골라내고, 데이터를 머신러닝에 사용할 수 있는 형태로
     바꾸는 과정이 주다.
	 
7. 학습을 수행하고 파라미터를 튜닝한다.
   - 실제로 학습을 수행하는 단계이다. 
   - 학습 알고리즘을 앞 단계에서 정했으나, 머신러닝 알고리즘을 조정하는 파라미터를
     시행착오를 거쳐가며 수정해보고 결과가 더 나은 값을 찾는 과정이다.
   - 사람이 직접 부여한 정답이나 규칙 기반으로 예측한 성능을 기준치로 하여 이를 뛰어넘는
     것을 목표호 삼는다.
	 
8. 시스템에 통합한다.
   - 머신러닝 기능을 기존 시스템에 통합하는 일을 수행하는 단계이다.
   - 주의할 점은 예측 성능과 그에 따른 비지니스 성과를 모니터링하는 것이다.
   
- 머신러닝 시스템 개발은 시행착오를 반복하는 과정
  위의 과정중 4 ~ 7번 과정을 반복해서 시험해보고 수정한다고 보면 된다.
  
- 머신러닝 이론 연구는 4 ~ 7번 과정을 중심으로 다루는 반면
  비지니스에서는 1 ~ 8번 전 과정을 다룬다.

* 위의 과정을 크게 묶었을때

(1) 해결하려는 문제를 머신러닝으로 풀 수 있는 문제로 바꾸기( 1, 2 )
(2) 문제를 풀기 위한 도구 선택 및 전처리( 3, 4, 5, 6 )
(3) 모델 구축하기( 7 )
(4) 기존 서비스에 통합하기( 8 )

* (1), (2) 과정( 문제 정의 및 전처리 )이 중요하다. 데이터가 아무리 많아도
  적절히 전처리하지 않으면 성능이 나오지 않는다.
  해결하려는 문제 자체가 사람도 하기 어려운 일이라면 머신러닝으로 해결하는 것
  역시 어렵다. 즉, 사람이 답을 낼 수 없는 문제라면 기계가 풀도록 할 수도 없다.
  
* 머신러닝으로 해결한 사례를 찾으면 먼저 다음 세 가지 사항을 검토한다
  1. 어떤 알고리즘을 사용했는가?
  2. 어떤 데이터를 특징으로 사용했는가?
  3. 머신러닝 부분을 어떻게 통합했는가?
